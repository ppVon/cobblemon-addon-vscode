Davrial's Fake Cobblemon Wiki Page: Custom Moves Detailed Guide + Breakdown

Preface

I, Davrial, wished to put out a thorough guide on the various different elements of a custom move. I applied to be a wiki editor for this exact reason, but was turned down (very politely, i have no ill will to them at all). But unfortunately my brain wouldnt let me just drop the idea, so here i am making a custom wiki page the hard way. This guide will break down all the possible elements contained within a move, ALL of them, to the best of my ability.


Folder/File Structure

When adding a custom move to your pack, the location it goes in is as such: [Pack]/data/cobblemon/moves/[movename].js. For custom abilities, the same structure is used but just replace /moves/[movename].js with /abilities/[abilityname].js


HTML Complete ZIP vs just the HTLM File

If you opened this from the zip with the folder of other contents, you will see that it has the exact same formatting as the actual wiki. The reason it needed to be in a ZIP is because much of the visual data is stored in separate javascript files and cannot be merged into the HTML file. This guide will work perfectly fine without those files, as you might be seeing if you didn't grab the ZIP and just grabbed the file, it will just be missing the vast majority of the visual appeal. If you got the ZIP, you can just grab the file instead if you want. If you grabbed the file unsure about the contents of the ZIP, now you know, and can make an informed decision on which you prefer. I dont care either way ive just got brainworms about this so i wanted it to look as accurate as possible.


Step 1: Basic Example Move - Tackle


We'll start basic: the move Tackle for an example. Tackle is a basic move that does damage and that's it, a great baseline for understanding the basics before moving on to complex effects.

({
	num: 33,
	accuracy: 100,
	basePower: 40,
	category: "Physical",
	name: "Tackle",
	pp: 35,
	priority: 0,
	flags: {contact: 1, protect: 1, mirror: 1, metronome: 1},
	secondary: null,
	target: "normal",
	type: "Normal",
	contestType: "Tough",
})

Basic Example Breakdown

Now let's go through and break down all the different elements of that:

num: 33: The "num" field indicates which official move is being targeted in the showdown index. I have recieved conflicting information on whether or not custom moves should have custom numbers; the wiki page says no, but multiple people have given advice saying they do. If one does add a custom num to a custom move, give it a large number which is unlikely to conflict with other custom addons'/mods' moves. I personally am using 64000 as the starting index for my moves in my Mario inspired addon.

accuracy: 100: This is the move's accuracy. Valid numbers are 1-100, as well was "true", for moves that always succeed like ally-targeting boots or guaranteed-hit attacks.

basePower: 40: This is the move's Power, how strong the move is. It's not relevant for Status moves and should be set to 0 for those.

category: "Physical": The kind of move being used. Valid options are "Physical", "Special, and "Status".

name: "Tackle": Pretty self-explanatory, this is the name of the move. Note: this is both what's shown in battle AND used for code references, so make sure any custom move names are formatted how you want them to look in game, and then make sure you keep using that specific formatting every time you reference the move in code.

pp: 35: PP is short for Power Points. It is how many times the pokemon can use the move before needing to be refreshed in some way, either via an item usage or via using a healing machine.

priority: 0: This is the Priority of the move. Higher Priority moves go before lower priority moves, a pokemon's Speed only matters within the same Priority level. Possible options are whole numbers from +6 to -6, with +3 being the highest any canonical attacking moves go to.

flags: {contact: 1, protect: 1, mirror: 1, metronome: 1}: Flags are very important and define a lot of various different things, primarily involving the move's interactions with other moves and abilities. There's many different flags, and we'll go over them all later, but for now we'll just cover the ones present here. The flags here define that: the move makes contact; the move can be blocked by Protect; the move can be copied by the move Mirror Move; and the move can be called and used by the random-move move Metronome.

secondary: null: This field is for if the move has secondary effects. This is primarily relevant for moves which are primarily damage with a separate effect that also happens not as part of the damage, such as having a chance to flinch or burn. Chance-based effects are usually secondary effects. Tackle does not have one, so it's set to null, important to note that it's null instead of not having the field entirely.

target: "normal": This is what determines which pokemon can be targeted by a move. The "normal" targeting method as shown here is 1 adjacent enemy or ally. There are many different target options, which i will go into later, but "normal" and "self" are the two most common by far.

type: "Normal": Pretty self-explanatory, this is what pokemon elemental Type the move is.

contestType: "Tough": And finally, this is what the contest appeal type of the move is. Currently, this serves no purpose in Cobblemon, but every move from before the generations where the official games removed them has one. Tackle here is Tough. It's not a required field as newer moves dont have it either. If you wish to have the field anyway, the possible options are: Beautiful, Clever, Cool, Cute, and Tough.


Step 2: Basic Additional Effects


Ok now that we know the extreme basics, we're going to step it up a little. Next up I'm going list and explain all the basic additional effects (that I could find by manually combing thru the code). What do I mean by "basic additional effects"? I don't know if there's an official term for them, but what I mean are things that don't require specialized coding, just basic effects with basic fields, such as increased crit rate.

isNonstandard: "Past": Starting off with one which is not relevant to custom moves, isNonstandard denotes a move isnt acquirable for one reason or another. The existing values are: "Past", which means the move was standard in the past but nolonger is as of the most recent pokemon games; "Unobtainable", which means for one reason or another the move isn't obtainable via any regular method (usually moves only story encounter pokemon have, or special event-givaway moves); "Gigantamax", for moves which are unique to G-Max form signature attacks; "LGPE", which is short for Let's Go Pikachu / Eevee, which is for the unique moves that only Partner Pikachu and Partner Eevee (which are unique forms with different stats from regular members of their species) can learn, and which not present outside of those games in official pokemon games; and "CAP", which is short for Create-A-Pokemon, the only 2 moves with this tag are signature moves from SMOGON's Showdown Create-A-Pokemon project.

forceSwitch: true: This effect forces the opponent pokemon to swap out, such as with moves like Roar and Whirlwind. If the fight is a wild battle, the battle is fled; if the fight is a trainer battle, the opposing pokemon is swapped out for a different one at random.

recoil: [1, 4]: This field denotes any recoil damage the move does. The format is similar to others that come later, but functions differently. In this case, the format is actually a fraction, specifically a fraction of the damage dealt, in this case 1/4th. So for example a move with this recoil which did 100 damage to the target would then do 25 damage in recoil to the user.

struggleRecoil: true: A directly related stat, "struggleRecoil", is used by the move... well, Struggle. Instead of doing recoil damage based on how much damage the target took, struggleRecoil instead does 1/4 of the user's max HP.

mindBlownRecoil: true,: Another different form of recoil, this one does damage based on a custom-code math formula. However, the only 2 official moves that use this (Mind Blown and Steel Beam) have the same formula, as they both do 1/2 of the user's max HP in recoil damage. Since the custom code is extremely simple, I'll make an exception and put it here, as this effect doesnt work without the accompanying code: 

onAfterMove(pokemon, target, move) {
		if (move.mindBlownRecoil && !move.multihit) {
			const hpBeforeRecoil = pokemon.hp;
			this.damage(Math.round(pokemon.maxhp / 2), pokemon, pokemon, this.dex.conditions.get('[Move Name Here]'), true);
			if (pokemon.hp <= pokemon.maxhp / 2 && hpBeforeRecoil > pokemon.maxhp / 2) {
				this.runEvent('EmergencyExit', pokemon, pokemon);
			}
		}
	}

hasCrashDamage: true: A related effect similar to recoil, Crash Damage is instead damage dealt to the user only upon missing a move, based on the user's max HP. This is used for high-risk-high-reward moves such as High Jump Kick. Just like with the previous example all existing moves have the same formula, and as with mindBlownRecoil, this does not function without it's accompanying code, so I'll make another exception and put it here: 

onMoveFail(target, source, move) {
		this.damage(source.baseMaxhp / 2, source, source, this.dex.conditions.get('[Move Name Here]'));
	}

status: 'slp': The basic "status" effect gives the target the status listed, always listed in a three-letter shortening. In this example, the target is put to sleep. Important note: status refers to the statuses that display on a pokemon's health bar UI in game, such as sleep, poison, or burn. A pokemon can only have 1 status effect at a time, and will not gain a new one if it already has one. I will list all possible options for this field later.

volatileStatus: 'confusion': Status-like effects that are not displayed are classified as "volatileStatus" instead. The example here being Confusion. Volatile Statuses get cleared when the pokemon swaps out, unless the effect is causing them to be unable to swap out (like being bound by Bind) Important note: While a pokemon can only have 1 status effect at a time, a pokemon can be affected by any number of different volatile statuses at the same time. I will list all possible options for this field later.

sideCondition: 'lightscreen': Another different type of status condition, sideCondition compromises effects that influence a whole side of the field, in this case Lightscreen.

boosts: { atk: 1, }: This is the effect you use to boost the user's stats. This basic example would raise the user's attack by 1 stage. This is also the effect you use to LOWER their stats, by setting the value to negative.

drain: [1, 2]: This effect is how much of the damage dealt by a move is returned as HP, as with Absorb and Leech Life. Like with Recoil, it's represented as a fraction ([1, 2] = 1/2 = 50% of the damage returned as health)

heal: [1, 4]: Similar to drain, but for purely healing effects. This is again a fraction calculation, but instead it's a fraction of the target's max HP, since no damage is being dealt unlike with drain effects. Example: Life Dew.

ohko: true: This is for One-Hit Knock-Out moves such as Horn Drill and Fissure.

critRatio: 2 This effect boosts the crit chance of the move it is on, as opposed to boosting the crit chance of later moves or boosting the user's overall crit chance.

multihit: [2, 5]: Multihit is for moves that hit multiple times, as you might be able to guess. There's two different ways to use it: "multihit: 2" for example would mean the move would always hit twice. Meanwhile, the example shown here, which is far more common, means that a move will hit 2 to 5 times, randomly. (Note that that means the bracketed numbers do NOT act like a fraction as with prior effects listed here)

stallingMove: true: This line is used for moves used to stall out turns, such as protective moves like Protect or healing moves like Rest.

selfSwitch: true: This effect is used for moves which swap the user pokemon out. Valid values are: true (note: no single nor double quote marks); 'copyvolatile', which is how Baton Pass works; and 'shedtail', which is how Shed Tail works.

selfdestruct: "always": This is what's used to faint your pokemon when they use the move, like with Explosion. Valid values: are "always", always faints the mon; and "ifHit", which only faints them if they successfully land the hit (as with Final Gambit).

isZ: "psychiumz": This field indicates a move is a Z-Move, and which Z Crystal needs to be held to activate it. No existing moves seem to do it, but based on the code it's theoretically possible to use a mismatched move and crystal to trigger a unique Z-Move, like having a Water move need a Firium Z to activate a custom move named "Steam Torrent" or something.

zMove: {basePower: 180}: This field denotes what additional effect the move has when boosted by being used as a Z-Move, if it's not converted into a unique Z-Move. There's various different effects that can be used, but this example simply boosts the move's base power to 180.

maxMove: {basePower: 100}: Similar to the previous field, this one just sets the power of a move when used as a Max move. This is primarily present on moves which typically have varying Power, or multihit moves, which lose those traits when they become Max Airstream or any other Max move.

ignoreAbility: true: This causes the move to ignore all relevant abilities on the target, or at least ones that are able to be ignored.

ignoreImmunity: true: This line is used for moves that ignore a target's type immunities.

ignoreDefensive: true: This line is used for moves that ignore a target's defense boosts.

ignoreEvasion: true: This line is used for moves that ignore a target's evasiveness boosts.

And finally, one last example before we leave this section, saved because it's the only one that goes in the secondary field: 

secondary: {chance: 10,status: 'brn',}
The "chance" field, very simply, makes a secondary effect happen with a random chance, instead of guaranteed. In this specific example, the move would have a 10% chance to burn the target.


Step 3: Advanced Additional Effects

Alright, now we're about to get into the weeds. This section will cover multiple common move effects that require custom code, as well as some less-common-but-simple ones. To be clear, by "custom code", i mean code that specifically is written out to cause an exact effect based on a given trigger. With some of the more common ones it'd be easy enough to copy-paste into your own custom move, without having to actually change any of the code. Others, custom code editing would be needed. The format of this section will be the code followed by the text describing what it does.


Freeze-Dry: Additional Super-Effectiveness

   onEffectiveness(typeMod, target, type) {
     if (type === "Water")
       return 1;
   }

This is the code block from Freeze-Dry, which makes a move supereffective against a given type that it wouldn't be normally, in this case making Water-type pokemon get hit supereffectively.


Flying Press: Addition Move Type

onEffectiveness(typeMod, target, type, move) {
	return typeMod + this.dex.getEffectiveness('Flying', type);
},

As you could figure based on using the same starting field syntax, this type of effect is similar but different. Instead of making the move supereffective against a specific target like the previous example, this instead adds a whole extra type's effectiveness to the move. This effect is currently only used in Hawlucha's Flying Press.

Targets

This one's real simple, in this section I'll just be going over all the possible target: values and what each one does. For the sake of readability, presented without additional commentary (with one exception for clarification).

Single Target:

target: "self": Targets the user.

target: "adjacentAlly": Targets 1 adjacent ally, cannot target the user.

target: "adjacentAllyOrSelf": Targets 1 adjacent ally or the user.

target: "normal" Targets any 1 adjacent pokemon, be it ally or foe.

target: "randomNormal" As with "normal", but targets any 1 adjacent pokemon at random instead of being decided by the trainer (Example: Thrash).

target: "any" As with "normal", except "any" is not restricted to adjacency. This is only different from "normal" in triple battles, rotation battles, and hoard battles.

target: "scripted": This target value is used in special moves with specific code which causes the user to target based on certain parameters. This is almost exclusively used to target the last pokemon to damage the user, as with the move Counter.

Multi-target:

target: "all": Targets all pokemon on both sides, including the user. Important Note: No existing damaging moves can target all, and trying to do so with a custom move breaks it, ending up only targeting the user and their allies and not any foes (I know from trying myself)

target: "allAdjacentFoes": Targets all adjacent foes.

target: "allAdjacent": Targets all adjacent pokemon, ally and foe alike, but not the user.

target: "allySide": Targets the user and adjacent ally pokemon

target: "allyTeam": As with "allySide", but also targets non-adjacent ally pokemon. This is only different from "allySide" in triple battles.

target: "foeSide": Targets all foes, including non-adjacent ones. Note: This the opposite of the difference between "allyTeam" and "allySide".


Flags


Common Flags:

First, here's a bunch of flags that are extremely common, many being basically omnipresent for certain kinds of moves, and when they're absent it's a purposeful move design decision.


protect: The move can be blocked by Protect, and similar moves.

mirror: The move can be copied by Mirror Move.

metronome: The move can be randomly called by the random-move move Metronome.

allyanim: Not related to the mechanical effects of the move, but instead the graphics. this flag makes it so that the user turns to look at their ally. Obviously used for most ally-targeting moves.

contact: The move has the user make physical contact with the target.

snatch: Used with moves that give boosts, this flag enables the move Snatch to steal the related boosts. Most boosting moves have this flag, though not all.

reflectable: The move can be reflected back by Magic Coat / Magic Bounce.


Move Category Flags:

Herein are various flags which indicate move categories, such as sound and wind moves, as well as related flags like recharging or healing moves.


bite: The move is considered a biting move. These moves are boosted by the ability Strong Jaw.

bullet: The move is considered a bullet, ball, or bomb move. These moves are resisted by the ability Bulletproof.

dance: The move is considered to be a dancing move. These moves are interacted with by the ability Dancer.

defrost: The move will un-Freeze the user when used, or un-Freeze any target hit with it.

explosive: The move is considered to be a explosive move. These moves are prevented by the ability Damp. [NOTE: Technically the flag doesnt exist in Cobblemon / Showdown. Its effects are instead hardcoded. However I highly recommend using this flag for custom moves and abilities, I know I will be using it for my Mario enemies addon, and at some point I plan on putting out an additions/tweaks addon that fixes stuff like that.]

powder: The move is considered a powder or spore move. Grass Type pokemon, pokemon with the Overcoat ability, and pokemon wearing the item Safety Goggles are immune to these moves.

pulse: The move is considered an aura or pulse move. These moves are boosted by the ability Mega Launcher.

punch: The move is considered a punching move. These moves are boosted by the ability Iron Fist and the item Punching Glove.

slicing: The move is considered a sharp slicing or piercing attack. These moves are boosted by the ability Sharpness.

sound: The move is considered to be Sound Based, and interacts with Soundproof and other sound-related effects.

wind: The move is considered to be Wind Based, and interacts with Wind Rider and other wind-related effects.


futuremove: Thee move does its damage, or main effect, on a future turn, independant of the user taking other actions in the meantime. See: Future Sight.

heal: All moves that heal, either plainly or via drain, have this flag, and are interacted by certain effects.

pledgecombo: The move combines with other pledge moves for unique effects.


charge: The move takes a turn before the main effect. This includes both actual charge moves like Solar Beam, but also moves like Fly or Dig which have a semi-invulnerable turn before attacking.

recharge: The move takes a turn to recharge after use, such as with Hyper Beam.

cantusetwice: This flag makes it so a move cannot be used twice in a row, such as with Blood Moon.


Exception and Failure Flags:

There's quite a few flags that exist to indicate a move cannot do something, does something normally disallowed, or prevents other moves from interacting with it. Here they are.


bypasssub: The move bypasses Substitute. Mostly used for beneficial ally-targeting effects.

distance: The move is considered to be attacking from long range or striking quickly from any distance away. Most of these moves also have the target of any instead of normal, not being bound by adjacency.

failencore: The move cannot be affected by Encore and similar effects. Usually moves that take multiple turns, regardless of it's a charge move or a recharge move.

nosleeptalk: The move cannot be called by Sleep Talk. Often moves that would damage the user or take multiple turns.

noassist: The move cannot be called by an ally pokemon's Assist.

failcopycat: The move cannot be copied by Copycat.

failmimic: The move cannot be copied by Mimic.

failinstruct: The move cannot be retriggered by an ally using Instruct, or similar moves. Usually two-turn moves.

failmefirst: The move cannot be copied and used first by another pokemon's Me First.

nonsky: This flag makes it so the move cannot hit pokemon that are in the air, such as FLying Types or pokemon with Levitate.

gravity: This flag makes it so the move cannot be used while the psuedo-weather Gravity is in effect. This includes most moves that involve bouncing, jumping, or flying.

noparentalbond: The move cannot be used a second time by Mega Kangaskan's Parental Bond ability.


Status Conditions and Volatile Conditions


Status Conditions

Format: "Name (nam): description"; the full name of the Status, the three-letter name for it in parentheses, then the description of what it does.

Burn (brn): Burn does 1/16 of the pokemon's Max HP in damage at the end of their turns, and a Burned pokemon deals half damage with Physical moves. Burn does not go away on its own. Fire Type pokemon cannot be Burned.

Paralysis (par): Paralysis causes a pokemon to have a 1 in 4 (25%) chance of being unable to act for a turn, and the pokemon's speed is halved. Paralysis does not go away on its own. Electric Type pokemon cannot be Paralyzed.

Sleep (slp): Sleeping pokemon cannot act, with special exceptions such as Sleep Talk. A pokemon put to sleep by its own move, such as with Rest, will be asleep for 2 turns. A pokemon put to sleep by other means will wake up in 2-5 turns.

Freeze (frz): Frozen pokemon cannot act, unless the chosen move would remove the Frozen condition, such as many Fire Type moves or Scald. A Frozen pokemon hit with such moves will also be unfrozen. Each turn a Frozen pokemon has a 20% chance to break out of being Frozen. This means that an especially unlucky pokemon could theoretically never unfreeze. Ice Type pokemon cannot be Frozen, and pokemon cannot be frozen in Harsh Sunlight.

Poison (psn): Poison does 1/8 of the pokemon's Max HP in damage at the end of their turns. Poison does not go away on its own. Both Poison Type and Steel Type pokemon cannot be Poisoned.

Badly Poisoned aka Toxic (tox): As with Poisoned, except the damage for Toxic starts at 1/16 of Max HP, and increases by an additional 1/16 each turn the pokemon is on the field. (So 1/16, 2/16, 3/16, etc.)


Volatile Statuses (also just called "Volatiles")

There are far too many preexisting Volatiles to go over in detail. Instead, I will go over a few extremely common and useful ones. Additionally, know that it is fairly easy to add your own volatiles within a custom move, as opposed to brand new Status Conditions which are extremely difficult and possibly impossible to do without full on modding.

Common Volatiles:

confusion: Easily the most common volatile status. Confuses the pokemon. You know what confusion is. If you dont, makes it so the pokemon has a chance to fail to use a move and hit themselves instead.

flinch: Makes the pokemon flinch, failing whatever move they were going to do.

charge: A volatile applied for the first turn of a move that requires charging.

mustrecharge: A volatile applied after the first turn of a move that requires recharging.

protect: The Protect effect. Can also be used with other moves which do the exact same effect without any additional effects, such as with Detect but not Baneful Bunker.

foresight: Makes the user's next attack ignore evasiveness and allows it to bypass Ghost's immunities. Used by Foresight and other moves that do the same.

substitute: Makes the user have a substitute. Note that this doesn't work on its own unlike some other volatiles, as it needs other data to determine the substitute doll's health.

partiallytrapped: Many moves trap the target on the field for a few turns. This is the volatile that is applied to the target when done so.

Other Volatiles?

There are many other volatiles. I even gathered up a list of all of them. There's 82 of them, to be precise. I had not counted all them until i got to making this part of this guide. I have decided i will not be listing all them because that feels like a waste of time and also genuinely feels like it wouldnt actually be useful information. So i'll just say this: If a move or ability has an ongoing effect, it's probably in the form of a volatile status.
